# Configuration Reference

Config file: `~/.innie/config.toml`

Generated by `innie init`. All values have defaults and the file is not required.

---

## Full Config Reference

```toml
# ~/.innie/config.toml

[user]
name = "Josh"               # Your name (injected into session context)
timezone = "America/Chicago"

[defaults]
agent = "innie"             # Active agent when INNIE_AGENT not set

# ── Embedding ─────────────────────────────────────────────────────────────────

[embedding]
provider = "docker"         # docker | external | none
model = "bge-base-en"       # Model name passed to embedding API

[embedding.docker]
url = "http://localhost:8766"  # Local embedding service (docker compose up)

[embedding.external]
# url = "https://api.openai.com/v1"
# api_key_env = "OPENAI_API_KEY"    # Env var name (never hardcode keys)
# model = "text-embedding-3-small"

# ── Heartbeat ─────────────────────────────────────────────────────────────────

[heartbeat]
enabled = false             # Enable automatic scheduled heartbeat
interval = "30m"            # Cron-style or duration string
model = "auto"              # LLM for extraction ("auto" = configured default)
collect_git = true          # Include git log/diff in collection
collect_sessions = true     # Include session logs in collection

# ── Index ─────────────────────────────────────────────────────────────────────

[index]
chunk_words = 300           # Words per text chunk
chunk_overlap = 60          # Overlap words between chunks (20%)
chunk_markdown_aware = true # Split on markdown headers before word-windowing

# ── Search ────────────────────────────────────────────────────────────────────

[search]
query_expansion = false     # Generate alt query phrasing via LLM (adds ~1-2s)
expansion_model = "auto"    # "auto" = uses heartbeat.model and heartbeat.external_url

# ── Context ───────────────────────────────────────────────────────────────────

[context]
max_tokens = 2000           # Max characters injected at session start

# ── Git ───────────────────────────────────────────────────────────────────────

[git]
auto_commit = false         # Auto git commit data/ after heartbeat
auto_push = false           # Auto git push after commit (requires remote)
```

---

## Config Sections

### `[user]`

| Key | Type | Default | Description |
|---|---|---|---|
| `name` | string | `""` | Your name, injected into session context |
| `timezone` | string | `"UTC"` | Your timezone (e.g., `"America/Chicago"`) |

### `[defaults]`

| Key | Type | Default | Description |
|---|---|---|---|
| `agent` | string | `"innie"` | Default active agent when `INNIE_AGENT` not set |

### `[embedding]`

| Key | Type | Default | Description |
|---|---|---|---|
| `provider` | string | `"docker"` | `docker` \| `external` \| `none` |
| `model` | string | `"bge-base-en"` | Embedding model name |

**provider = "docker"** — runs local `bge-base-en` via Docker on port 8766. Zero cost, no API key. Run `docker compose up -d embeddings` first.

**provider = "external"** — any OpenAI-compatible embedding endpoint.

**provider = "none"** — keyword-only search. No vectors, no Docker required.

### `[embedding.docker]`

| Key | Type | Default | Description |
|---|---|---|---|
| `url` | string | `"http://localhost:8766"` | Embedding service URL |

### `[embedding.external]`

| Key | Type | Default | Description |
|---|---|---|---|
| `url` | string | — | OpenAI-compatible embeddings endpoint |
| `api_key_env` | string | — | Env var name containing API key |
| `model` | string | — | Model name to request |

### `[heartbeat]`

| Key | Type | Default | Description |
|---|---|---|---|
| `enabled` | bool | `false` | Enable automatic heartbeat on schedule |
| `interval` | string | `"30m"` | How often to run (duration or cron) |
| `model` | string | `"auto"` | LLM model for extraction phase |
| `collect_git` | bool | `true` | Include git log/diff in Phase 1 |
| `collect_sessions` | bool | `true` | Include session logs in Phase 1 |

### `[index]`

| Key | Type | Default | Description |
|---|---|---|---|
| `chunk_words` | int | `300` | Target words per chunk |
| `chunk_overlap` | int | `60` | Overlap words between adjacent chunks (20%) |
| `chunk_markdown_aware` | bool | `true` | Split on `##`/`###` headers before word-windowing |

**Tuning:** Smaller chunks = more precise retrieval but less context per result. 300/60 targets the useful embedding range for BGE-base-en (~225 tokens). Set `chunk_markdown_aware = false` to restore pure word-window behavior.

### `[search]`

| Key | Type | Default | Description |
|---|---|---|---|
| `query_expansion` | bool | `false` | Generate one alt phrasing via LLM before searching |
| `expansion_model` | string | `"auto"` | `"auto"` uses `heartbeat.model` and `heartbeat.external_url` |

**query_expansion:** Bridges vocabulary mismatch (e.g., searching "login" when content says "authentication"). Adds ~1-2s latency. Silent no-op if the LLM endpoint is unavailable.

### `[context]`

| Key | Type | Default | Description |
|---|---|---|---|
| `max_tokens` | int | `2000` | Max characters from search results injected at session start |

### `[git]`

| Key | Type | Default | Description |
|---|---|---|---|
| `auto_commit` | bool | `false` | `git add -A && git commit` in `data/` after each heartbeat |
| `auto_push` | bool | `false` | `git push` after commit (requires remote configured) |

!!! warning "git.auto_push"
    Only enable `auto_push` if you have a remote configured and appropriate credentials. innie does not manage git credentials.

## Dot-Notation Access

Config values can be read in code with dot notation:

```python
from innie.core.config import get

provider = get("embedding.provider", "docker")
model = get("heartbeat.model", "auto")
commit = get("git.auto_commit", False)
```

## Multiple Agents, Single Config

All agents share the same `config.toml`. Agent-specific behavior is controlled by:

1. `INNIE_AGENT` env var or `defaults.agent` in config — selects active agent
2. `HEARTBEAT.md` — per-agent extraction instructions
3. `SOUL.md` — per-agent identity
4. Per-command `--agent` flag
